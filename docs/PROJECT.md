# Project Overview

## ชื่อระบบ

BaanBook

## คำอธิบายสั้น

BaanBook คือ personal home management system สำหรับเจ้าของบ้านหรือครอบครัวที่ต้องการเก็บข้อมูลทุกอย่างเกี่ยวกับบ้านไว้ในระบบเดียว ตั้งแต่ค่าใช้จ่าย เอกสาร งานซ่อม Warranty รีโนเวท เครื่องใช้ไฟฟ้า Shopping และ Mortgage

## ปัญหาที่ต้องแก้

ข้อมูลเกี่ยวกับบ้านมักกระจัดกระจายอยู่หลายที่ เช่น ใบเสร็จในแชท รูปถ่ายในมือถือ เอกสารใน Google Drive รายการซ่อมในสมุดจด ค่าใช้จ่ายใน spreadsheet และ Warranty ที่จำไม่ได้ว่าหมดเมื่อไร เมื่อเวลาผ่านไปข้อมูลสำคัญหาย ค้นยาก และไม่สามารถเห็นภาพรวมต้นทุนของบ้านได้

BaanBook แก้ปัญหานี้ด้วยการทำให้บ้านมี record กลางที่ค้นได้ เชื่อมโยงกัน และดูย้อนหลังได้

## เป้าหมายหลัก

- เก็บข้อมูลบ้านแบบเป็นระบบ
- ลดการลืมงาน maintenance และวันหมด warranty
- เห็นภาพรวมค่าใช้จ่ายบ้านทั้งรายเดือน รายปี และตามหมวด
- เก็บเอกสารสำคัญพร้อม metadata เพื่อค้นง่าย
- สร้าง timeline ประวัติบ้านที่ช่วยตอบคำถามย้อนหลังได้
- ช่วยวางแผน shopping, renovation และ mortgage
- รองรับการใช้งานส่วนตัวก่อน แล้วขยายเป็นครอบครัวหรือหลายบ้านได้

## กลุ่มผู้ใช้

### เจ้าของบ้าน

ผู้ใช้หลักที่ต้องการเก็บข้อมูลทุกอย่างเกี่ยวกับบ้านตัวเอง เช่น ค่าไฟ งานซ่อม เครื่องใช้ เอกสาร และหนี้บ้าน

### คู่สมรสหรือสมาชิกครอบครัว

ผู้ใช้ร่วมที่ต้องการเห็นข้อมูลเดียวกัน เพิ่มข้อมูลบางส่วน หรือช่วยดูแลบ้านร่วมกัน

### เจ้าของบ้านหลายหลัง

กลุ่ม future target ที่ต้องการจัดการบ้านมากกว่าหนึ่งหลัง เช่น บ้านหลัก คอนโด บ้านปล่อยเช่า หรือบ้านต่างจังหวัด

## Scope ของระบบ

### In Scope

- จัดการบ้านและพื้นที่ภายในบ้าน
- จัดการค่าใช้จ่าย
- จัดการรายการรีโนเวท
- จัดการเครื่องใช้ไฟฟ้าและทรัพย์สินสำคัญ
- จัดการเอกสารและไฟล์แนบ
- จัดการ maintenance และ reminder
- จัดการ warranty
- สร้าง timeline กิจกรรมของบ้าน
- จัดการรายการ shopping และ wishlist
- จัดการข้อมูล mortgage
- Dashboard ภาพรวม

### Out of Scope สำหรับช่วงแรก

- ระบบ marketplace
- ระบบจ่ายเงินในแอป
- OCR อัตโนมัติเต็มรูปแบบ
- Integration กับธนาคาร
- Mobile native app
- IoT integration
- Accounting ขั้นสูง
- ระบบสำหรับนิติบุคคลหรือหมู่บ้าน

## Product Positioning

BaanBook อยู่ระหว่าง personal finance, home inventory, maintenance tracker และ document vault แต่จุดแข็งคือ context ของบ้าน ข้อมูลแต่ละอย่างไม่ได้แยกกันอยู่ แต่ผูกกับบ้าน ห้อง item เอกสาร และ timeline

## Success Metrics

- ผู้ใช้เพิ่มข้อมูลบ้านแรกได้ภายใน 5 นาที
- ผู้ใช้บันทึกค่าใช้จ่ายหรือ maintenance ได้ภายใน 30 วินาที
- ผู้ใช้ค้นเอกสารหรือ warranty ได้ภายใน 10 วินาที
- Dashboard ตอบคำถามพื้นฐานได้ทันที เช่น เดือนนี้จ่ายอะไรไปบ้าง มี warranty อะไรใกล้หมด มี maintenance อะไรค้าง
- ข้อมูลสำคัญทุก record มี owner และ permission ชัดเจน

## Key Design Decisions

- เริ่มจาก web app ก่อน เพราะเหมาะกับ data entry และ dashboard
- ใช้ Supabase เป็น backend เพื่อลดเวลาสร้าง database และ storage
- แยก domain ของแต่ละ feature ให้ชัดเจน แต่ใช้ shared entities เช่น home, room, document, attachment และ timeline
- ใช้ shadcn/ui เป็น component foundation เพื่อให้ UI สม่ำเสมอและปรับแต่งได้
- ออกแบบ database ให้รองรับหลายบ้านและสมาชิกหลายคนตั้งแต่แรก แม้ MVP จะเริ่มจาก single owner

## Non-Goals

- ไม่สร้างระบบบัญชีเต็มรูปแบบ
- ไม่สร้างระบบบริหารทรัพย์สินระดับ enterprise
- ไม่ออกแบบให้ทุกอย่าง configurable ตั้งแต่วันแรก
- ไม่เพิ่ม abstraction ที่ยังไม่มี use case จริง
